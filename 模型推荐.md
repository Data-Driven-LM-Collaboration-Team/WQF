# **2025.12.17** 
## 任务定义
基于GLEMOS提供的性能矩阵（300多的模型架构+400多的图数据集）
在给定一个图学习任务的前提下，直接预测一个性能最优或近似最优的 GNN 模型结构，而不在推理阶段进行任何 GNN 的训练或验证，并且推理阶段不能做RL、搜索、进化。 

**本质上是学习一个“经验函数”：根据以往大量任务–模型–性能的记录，学会在新任务上直接做决策。**

# **2025.12.25** 
## 任务定义 GNN架构推荐
GLEMOS、MetaGL 局限于有限模型集合，数量只有300-400，并且依赖性能矩阵
NAS：模型搜索空间大，耗时间耗资源 

## Pipeline
- SFT：定义模型架构空间（参考NAS-bench-graph） 输入：图数据集的自然语言描述 输出：一个完整的、符合约束的架构表示
- RL/RM:引入模型性能信号，目的是在不破坏结构合法性的前提下，让 LLM 更倾向生成 validation performance 更高的架构。
